Paper tecnico en LaTeX listo para Overleaf - Kevin Fernandez Sanchez (ESPOL) - Mayo 2026

EMG Trainer: una plataforma end-to-end para el entrenamiento y despliegue de modelos TinyML aplicados al control mioelectrico de protesis de mano

Kevin Fernandez Sanchez
FIEC - Escuela Superior Politecnica del Litoral (ESPOL), Guayaquil, Ecuador
kfernandezsanchez.anam@gmail.com - Mayo 2026
Resumen. El control mioelectrico de protesis de mano basado en aprendizaje automatico se ve limitado por la necesidad de re-entrenar modelos por usuario y por el coste computacional de inferir en tiempo real sobre microcontroladores. Se presenta EMG Trainer, una plataforma web full-stack que integra adquisicion BLE de senales EMG mediante sensores uMyo, una API en FastAPI con procesamiento asincrono via Celery, un pipeline de entrenamiento de modelos TinyML cuantizados y un controlador de actuadores Dynamixel para una mano antropomorfa. Resultados preliminares muestran latencias de inferencia inferiores a 20 ms en ESP32 y precisiones superiores al 90% sobre cinco gestos basicos.

Palabras clave: TinyML, EMG, ESP32, FastAPI, Dynamixel, cuantizacion.

1. Introduccion

La perdida de una extremidad superior impacta de manera severa la autonomia funcional. En Ecuador, el acceso a protesis mioelectricas comerciales es limitado por su costo. TinyML permite llevar inferencia ML a microcontroladores con memorias inferiores a 1 MB y consumos por debajo de 100 mW, habilitando clasificadores embebidos en la misma electronica que controla la protesis. EMG Trainer unifica adquisicion, etiquetado, entrenamiento, cuantizacion y despliegue en una plataforma operativa.

Contribuciones:

  1. Arquitectura modular Vue + FastAPI + Celery + Redis + PostgreSQL para sesiones de entrenamiento personalizadas.
  2. Pipeline reproducible de adquisicion BLE de EMG via uMyo y firmware ESP32.
  3. Entrenamiento de modelos TinyML con cuantizacion post-training a int8 y exportacion a binarios embebibles.
  4. Controlador de protesis Dynamixel acoplado al modelo en tiempo real con planificador y safety manager.

2. Trabajo relacionado

Las caracteristicas clasicas de Hudgins (MAV, ZC, SSC, WL) siguen siendo competitivas. TensorFlow Lite for Microcontrollers y Edge Impulse han popularizado el despliegue cuantizado en ARM Cortex-M y ESP32. EMG Trainer se diferencia al aportar una capa de orquestacion web y un hardware fisico de protesis basado en Dynamixel.

3. Arquitectura del sistema

El sistema se compone de cuatro subsistemas: hardware de adquisicion (uMyo + ESP32 BLE), backend asincrono (FastAPI + Celery + Redis + PostgreSQL), frontend web (Vite + Vue 3 + TypeScript) y protesis robotica (Dynamixel SDK).

El backend sigue un patron services / repositories / routers. Las tareas largas (train, test) se despachan a workers Celery via Redis, manteniendo la API reactiva.

4. Adquisicion de senal EMG

Firmware en ESP32 lee los modulos uMyo via BLE y replica los valores por serial. Frecuencia efectiva de muestreo ~33 Hz por canal, suficiente para envolvente de activacion en gestos estaticos. Cuando se requiere mas resolucion, el backend soporta fuentes serial directa y UDP.

#include <uMyo_BLE.h>

void setup() { Serial.begin(115200); uMyo.begin(); }

void loop() {
  uMyo.run();
  int dev_count = uMyo.getDeviceCount();
  for (int d = 0; d < dev_count; d++) {
    Serial.print(uMyo.getMuscleLevel(d));
    if (d < dev_count - 1) Serial.print(' ');
    else                   Serial.println();
  }
  delay(30);
}

5. Preprocesamiento y caracteristicas

Ventanas de N=64 muestras con solape 50%. Caracteristicas por canal: MAV, WL, ZC (con umbral epsilon), SSC y RMS. Por defecto 4 canales x 5 caracteristicas = 20 dimensiones, normalizadas con min-max por sesion.

MAV = (1/N) · Σ |x_i|,   RMS = sqrt((1/N) · Σ x_i²),   WL = Σ |x_i - x_{i-1}|

6. Modelo TinyML y entrenamiento

Red feed-forward: dos capas densas (32, 16) con ReLU y softmax de K clases. ~1500 parametros. Gestos: reposo, cierre completo, pinza fina, extension y apuntar. El endpoint POST /api/v1/tasks/train encola una tarea Celery que carga CSV, calcula features, entrena, cuantiza a int8 y exporta a TFLite. Persiste metricas en PostgreSQL.

q = round(x / s) + z,   x ≈ s · (q - z)

7. Plataforma web

Frontend Vue 3 + Vite con tres flujos: gestion de pacientes/dispositivos, captura de sesiones (WebSocket en vivo + etiquetado), entrenamiento/evaluacion y descarga del modelo .tflite para flashear al ESP32. API REST /api/v1/<recurso>, OpenAPI auto-documentado por FastAPI, autenticacion JWT (python-jose).

8. Control de la protesis

Mano Omnihand: 5 motores Dynamixel, uno por dedo. Modulos: hand_controller (gesto -> objetivo articular), group_motion (sync write), session_runner (registro de latencias/torque), safety_manager (limites de corriente y recorrido). Latencia extremo a extremo < 50 ms.

9. Resultados preliminares

1 sujeto sano, 4 sensores en antebrazo, 500 ventanas por gesto, 5 gestos. Split 80/10/10.

Caida por cuantizacion < 3 puntos porcentuales. Limitaciones: cohorte pequena, sensibilidad al reposicionamiento del sensor y ausencia de control proporcional (proximo trabajo).

10. Conclusiones

EMG Trainer demuestra la viabilidad de un sistema end-to-end de entrenamiento y despliegue TinyML sobre una protesis de bajo costo. Trabajo futuro: dataset con sujetos amputados y protocolo clinico formal, 1D-CNN cuantizada, control proporcional via regresion y telemetria remota.
